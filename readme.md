# 프로젝트: SV 덱 프로파일러

> [ 🎯 데이터를 기반으로 사용자가 '표준 덱'을 구축하도록 돕는 자동화 분석기 구현 프로젝트입니다.]

---

## 1. 문제 정의

* TCG(섀도우버스) 유저들은 메타를 참고할 때, 수십 개의 '덱 리스트'를 보게 됩니다.
* 하지만 이 리스트들은 '개인의 취향'이 반영되어, 어떤 카드가 '필수(고정)'이고 어떤 카드가 '선택'인지 많은 자료와 경험으로 축적시킨 '감'으로 판단해야 하는 '피로도'가 있었습니다.

## 2. 해결책

이 프로젝트는 '감'이 아닌 '수학적 통계'로 이 문제를 해결합니다.

1.  `Selenium`을 사용해 JavaScript로 동적 로딩되는 '[svlabo](https://svlabo.jp/)'의 덱 리스트 페이지에 '자동으로 방문'합니다.
2.  `BeautifulSoup`로 '로딩이 완료된' HTML을 '자동 파싱'하여, '모든 카드'의 '이름'과 '덱별 사용량'을 '자동 추출'합니다.
3.  추출한 데이터를 '자동 분석'하여 각 카드의 '평균 사용량'을 계산합니다.

## 3. 핵심 기술

* **Language:** `Python`
* **Web Scraper:** `Selenium`, `BeautifulSoup4` (동적 JavaScript 페이지 로딩, HTML 데이터 추출)

## 4. 실행 방법

1.  필요한 라이브러리를 설치합니다.
    ```bash
    pip install flask selenium beautifulsoup4 webdriver-manager
    ```
2.  파이썬 스크립트를 실행합니다. (터미널에서)
    ```bash
    python app.py
    ```
3.  터미널에 `* Running on http://127.0.0.1:5000` 메시지가 뜨면,
4.  '웹 브라우저'를 열고 `http://127.0.0.1:5000` 주소로 '방문'합니다.
5.  약 10초간 '자동 크롤링'이 실행된 후, '실시간 분석 결과'가 '웹페이지'에 '표시'됩니다.

## 5. 다음 단계

* 포스트의 필터(덱타입, 레이트, 사용일, 연승수, 그룹 등)를 불러오기
* 필터에 따라 불러진 덱리스트를 레이트, 연승수, 사용일 등의 가중치를 더해 "평균 덱리스트 생성"
* 고정칸와 선택칸을 분리하여 선택칸의 선택지까지 제공
